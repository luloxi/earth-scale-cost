Assumptions

This is the demand assumption. The default is 8.2 billion people.
Default is 1 tx per person per day. Increase it to test more aggressive mass-adoption scenarios.

Live metric definition

Required TPS = population × daily tx per user ÷ 86,400

Earth-Scale multiplier = max(1, required TPS ÷ live TPS)

Earth-Scale Cost per Tx = live average fee × Earth-Scale multiplier

This keeps the whole ranking observable and updatable clientside with no placeholders.

Method and sources

This page intentionally favors live observability over theoretical promises. If a number cannot be refreshed in the browser, it does not belong in the ranking.

How the live inputs work

Most chains here use live TPS and average fee data from their Chainspect chain pages, fetched through api.codetabs.com so the browser can read them cross-origin.

XRP Ledger is currently sourced from Blockchair live stats. Unichain is currently sourced from Blockscout live stats.

The page refreshes every 30 seconds and reranks immediately when new values arrive or when you change the demand assumptions.

Caveats

This is a live practical benchmark, not a full cost-model simulation. Real capacity under extreme demand can differ from current observed TPS, and fee formation differs across architectures.

Still, it is already much more honest than static placeholder leaderboards because every row is tied to live, inspectable data.
